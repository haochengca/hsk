# HSK1 任务池 + 中断续做 功能头脑风暴与实施计划（v1）

## 1. 背景与目标
- 新增“任务”能力：支持直接创建“HSK1 级全部汉字”任务包，减少手动选字成本。
- 支持任务中断后可继续：用户下次进入时可从上次进度恢复，不必从头开始。
- 保持当前默写状态机与判定体验一致，避免引入新的学习路径割裂。

## 2. 需求拆解（用户视角）
### 2.1 任务创建
- 作为家长/老师，我希望一键选择“HSK1 全部汉字”，自动生成任务。
- 作为孩子，我希望看到该任务总字数、预计用时、完成条件。

### 2.2 执行与中断
- 作为孩子，我在任务进行中可以主动暂停/退出。
- 作为孩子，我重新进入时可以点击“继续任务”，从断点继续。

### 2.3 完成与复盘
- 任务完成后给出任务级总结（正确率、错字列表、薄弱字）。
- 已完成任务不可再“继续”，但可“再次挑战（重置）”。

## 3. 头脑风暴：关键产品方案

### A. 任务模板化（高优先级）
- 内置模板：`HSK1_ALL_CHARS`（来源：HSK1 全部汉字数据集）。
- 创建任务时仅需选择模板 + 截止时间（可选）+ 每次练习题量（可选）。
- 保留“自定义任务”入口，避免模板限制扩展性。

### B. 断点续做策略（高优先级）
- 任务进度最小单位：`currentIndex`（当前做到第几个字）+ `completedItems` + `wrongItems`。
- 自动保存时机：
  - 每次判题后；
  - 点击“下一个字”后；
  - 用户主动暂停时。
- 恢复规则：
  - 若任务状态为 `in_progress` 且存在快照，则展示“继续任务”；
  - 若快照损坏或版本不兼容，提示“从最近可用点恢复/重置任务”。

### C. 任务状态机（建议）
- `pending`（未开始）
- `in_progress`（进行中，可继续）
- `paused`（用户主动暂停，可继续）
- `completed`（已完成）
- `archived`（归档）

> 兼容当前默写状态机：任务状态机管理“任务级生命周期”，默写状态机管理“单次练习流程”。

### D. 数据模型草案
- `tasks`
  - `id`, `ownerId`, `assigneeId`, `templateType`, `status`, `createdAt`, `dueAt`
- `task_items`
  - `taskId`, `itemId`(汉字), `seq`, `sourceLevel`
- `task_progress`
  - `taskId`, `currentIndex`, `completedCount`, `correctCount`, `wrongItems`, `lastSnapshotAt`, `sessionId`

### E. 体验细节建议
- 任务卡片上突出“已完成 x/y、上次练习时间、继续按钮”。
- 若中断超过 3 天，进入任务前弹出“快速回顾 5 题”暖启动。
- 对未开始任务显示“预计 N 分钟”，降低心理门槛。

## 4. 技术方案草案

### 4.1 后端 API（建议增量）
- `POST /api/tasks`：创建任务（支持模板 `HSK1_ALL_CHARS`）。
- `GET /api/tasks/:id`：获取任务详情及进度。
- `POST /api/tasks/:id/start`：开始任务。
- `POST /api/tasks/:id/progress`：保存任务进度快照。
- `POST /api/tasks/:id/resume`：恢复任务，返回当前题目索引。
- `POST /api/tasks/:id/complete`：完成任务并结算。

### 4.2 前端改造点
- 任务列表新增“模板创建入口（HSK1 全部汉字）”。
- 任务详情页新增“继续任务”按钮与进度条。
- 默写提交后调用任务快照保存接口，保证断点可恢复。

### 4.3 兼容与迁移
- 对历史任务默认 `status=pending`，`currentIndex=0`。
- 对无进度快照的进行中任务，允许从首题恢复并提示。

## 5. 实施计划（Plan）

### Phase 0：需求定稿（0.5 天）
- 明确“HSK1 全部汉字”的精确集合与排序规则。
- 确认“继续任务”的恢复粒度（题目级 vs 字符笔画级）。
- 输出接口契约初稿。

### Phase 1：数据与接口（1~2 天）
- 增加任务模板解析器（支持 `HSK1_ALL_CHARS`）。
- 新增/扩展任务与进度 API。
- 增加服务端进度写入幂等保护（防重复提交）。

### Phase 2：前端流程（1~2 天）
- 新增“创建 HSK1 全汉字任务”交互。
- 新增任务详情中的“继续任务”入口。
- 在默写流程中接入任务进度实时保存。

### Phase 3：质量保障（1 天）
- 单元测试：模板解析、进度推进、恢复逻辑。
- E2E：创建任务 -> 做到一半退出 -> 继续 -> 完成。
- 回归：不带任务的普通默写流程不受影响。

### Phase 4：灰度与观测（0.5 天）
- 先对内部账号灰度。
- 观察指标：任务完成率、续做成功率、中断后放弃率。

## 6. 验收标准（DoD）
- 可一键创建 HSK1 全汉字任务。
- 用户中断后可继续，恢复位置误差为 0（题目级）。
- 任务完成后可查看任务总结，且不可再继续。
- 自动化用例覆盖“创建/中断/恢复/完成”主路径。

## 7. 风险与规避
- 风险：频繁保存进度导致接口压力。
  - 规避：前端节流 + 后端幂等 + 批量落盘。
- 风险：任务状态与默写状态冲突。
  - 规避：双状态机分层，建立单向事件流。
- 风险：HSK 数据源升级导致任务集合变化。
  - 规避：创建任务时快照 item 列表，避免后续漂移。

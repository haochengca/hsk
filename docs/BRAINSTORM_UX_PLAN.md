# HSK 网站功能与体验头脑风暴计划（v1）

## 1. 目标
- 提升用户完成默写闭环的成功率（开始默写 -> 完成判定 -> 复盘错题）。
- 降低关键流程中的误操作和状态困惑（按钮状态、提示时机、会话结束）。
- 增加留存驱动力（连续打卡、挑战反馈、成长可视化）。

## 2. 当前关键痛点（头脑风暴汇总）
- 默写页面按钮状态复杂，用户容易不理解“为什么按钮不可点”。
- 会话结束与最后一题边界场景多，容易出现提示丢失、按钮冲突。
- 学习/默写/错题之间跳转成本高，缺少“下一步建议”。
- 父母/孩子协作信息虽然有数据，但提醒和反馈仍偏弱。
- 统计数据齐全，但“用户可感知的成长叙事”不够强。

## 3. 产品改进方向

### A. 默写交互状态机（高优先级）
- 统一五态：未开始、预览中、答题中、已判定待下一题、已结束。
- 所有按钮（开始、重开、停止、完成并判定、下一个字）由同一状态控制器渲染。
- 每个状态给出明确提示文案与禁用原因。

### B. 结果反馈与复盘（高优先级）
- 判定后固定展示“结果卡”：正确/错误、得分、下一步建议。
- 增加“一键加入复习计划”而非隐式进入错题本。
- 最后一题结束页展示本轮总结（正确率、耗时、薄弱点）。

### C. 学习路径引导（中优先级）
- 学习页增加“推荐下一步”卡片（去默写、去错题、去写字）。
- 默写结束后提供 3 个快捷动作：错题复习、再次挑战、返回学习。

### D. 激励体系升级（中优先级）
- 挑战从固定阈值升级为动态目标（基于最近7天表现）。
- 徽章增加可视化进度（例如“一周坚持 5/7”）。
- 结算动画支持静音模式与简洁模式切换。

### E. 家庭协作体验（中优先级）
- 父母端增加“待跟进提醒”（孩子连续低正确率、任务逾期）。
- 任务支持优先级和截止时间，孩子端显示“今日最重要任务”。

### F. 管理与可运维性（中优先级）
- 审核页支持“批量操作预览”与撤销窗口（例如 10 秒可撤销）。
- 增加操作日志检索（按审核人、用户、时间）。

### G. 性能与可访问性（持续项）
- 首屏加载性能优化（列表虚拟化、静态资源分包）。
- 键盘可操作性和 ARIA 完整补齐。
- 移动端触控交互统一（点击区域、误触保护）。

## 4. 优先级路线图

## Phase 1（1-2 周）：稳定核心流程
- 默写按钮状态机重构。
- 判定结果卡统一。
- 最后一题结算页标准化。
- 新增相关 E2E：`review-state-machine.spec.js`。

## Phase 2（2-3 周）：增强体验闭环
- 学习页“下一步建议”卡。
- 默写结束快捷动作。
- 错题加入/移除确认优化。
- 新增 E2E：`review-finish-actions.spec.js`。

## Phase 3（2 周）：提升留存与家庭协作
- 动态挑战目标。
- 父母跟进提醒。
- 任务优先级/截止时间。
- 新增 E2E：`parent-followup-alerts.spec.js`。

## Phase 4（持续）：质量与性能
- 性能基线（LCP/TTI）建立。
- 可访问性审计修复。
- 稳定性回归矩阵持续扩展。

## 5. 指标体系（每周复盘）
- 体验指标：
  - 默写流程完成率（开始 -> 完成）
  - 单题误操作率（点击无效按钮次数）
- 学习效果指标：
  - 7日正确率变化
  - 错题回流后再错率
- 留存指标：
  - D1/D7 留存
  - 连续打卡中位天数
- 家庭协作指标：
  - 父母任务下发率
  - 任务按时完成率

## 6. 验证与测试计划
- 每个功能必须包含：
  - 行为验证（手工场景）
  - 权限验证（父母/孩子/管理员）
  - 回归验证（Playwright E2E）
- E2E 增量原则：
  - 每个 Phase 至少新增 1 条关键路径用例。
  - 所有线上 bug 先补回归用例再修复。

## 7. 风险与规避
- 风险：状态逻辑继续分散，修一个坏两个。
  - 规避：统一状态机函数、避免分散按钮赋值。
- 风险：激励过强影响学习质量。
  - 规避：目标动态校准，避免只刷次数不看正确率。
- 风险：父母提醒过多造成干扰。
  - 规避：提醒分级与可配置频率。
